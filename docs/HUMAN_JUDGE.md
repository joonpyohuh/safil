# SAFIL — Human Judge 결과

평가일: 2026-07-19  
대상: 카페 프로필 → 홍보 문구 생성 → 선택·복사 → 히스토리 재사용

## 평가 방식

실제 파일럿 사용자를 가정한 세 관점으로 동일한 핵심 여정을 정적 코드 감사했다.

1. 디지털 자신감이 낮은 40대 로스터리 카페 사장(Android, 한 손 사용)
2. 콘텐츠 결정 피로가 큰 30대 신규 카페 사장(iPhone)
3. 모바일 UX·접근성 감사(느린 네트워크와 Supabase/OpenAI 장애 포함)

사용자 평가는 아래 10개 항목을 각 10점으로 채점했다.

- 첫 30초 이해
- 프로필 설정 부담
- 모바일 조작성
- 생성 입력 간편성
- 로딩·오류 회복
- 결과 품질·구체성
- 이유 설명·신뢰
- 선택·복사 완료감
- 히스토리 재사용성
- 전체 완결성

통과 기준: **모든 평가가 90/100 이상**

## 반복 개선 내용

### 1차 — 끊긴 핵심 여정 복구

- 프로필 저장 후 문구 만들기로 자동 연결
- 미등록 사용자의 생성 진입 안내
- 실제 히스토리 목록, 재복사, 입력 재사용
- 복사·선택 실패와 성공 상태 분리
- 준비 중인 이미지·안내물 카드 비활성화
- 로딩·오류 경계 및 체험용 문구 품질 개선

### 2차 — 모바일 실패 복구와 결정 피로 개선

- 생성 timeout, 기존 결과 보존, 재생성
- 스크린리더용 상태 알림과 결과 초점 이동
- 큰 터치 영역, 키보드 대응, 고대비 focus
- 메뉴·최근 기록 기반 홍보 소재 추천
- 선택·복사 행동 단순화
- OpenAI·Supabase 일부 장애 시에도 결과 사용 가능

### 3차 — 개인화와 신뢰 강화

- 목적·채널별 길이, 구성, 행동 안내를 AI 프롬프트에 명시
- 이유에 실제 반영한 프로필·입력·표현을 설명하도록 요구
- 히스토리에서 당시 생성한 3안 비교
- 로스터리 강점·맛·산지·제조 방식 입력 유도
- 복사 성공과 기록 저장 실패를 독립적으로 처리

### 개편 핵심
- AI는 글자 없는 배경만 생성 → 한글은 캔버스 오버레이 (깨짐 원천 제거, 즉시 수정)
- 클라이언트 압축(HEIC→JPEG, EXIF 회전 보정)으로 업로드 실패 해결
- 비전 분석으로 사진 분위기·제목 자동 제안, 사진만 올리면 자동 생성
- 변형별 "사진 반영/새로 그림" 배지로 정직한 안내
- blob 사전 캐시 + share→download 폴백으로 iOS 공유 안정화

## 남은 위험

- 실제 iPhone/VoiceOver 및 저사양 Android 실기기 테스트는 별도 필요
- 히스토리가 많아지면 검색·날짜/채널 필터 필요
- 클라이언트 timeout 후 서버 요청이 완료되면 중복 생성 가능
- 인증 없는 단일 매장 API는 공개 서비스 전에 Supabase Auth와 rate limit 필요
